# 故障排查

## 状态图无法渲染

先运行 `java -version`。PlantUML PNG/SVG/PDF 需要 Java 11 或兼容 JRE，且
`docs/plantuml.jar` 必须随发布包存在。执行 `fcstm-gui --self-check` 可区分 Java、
JAR、子进程和输出图片问题。

## Linux 无法启动 Qt

发布包已携带项目所需 Qt 与关键 X11/GL 动态库，但宿主仍需兼容的 glibc 和显示
环境。无桌面环境验收时使用 `QT_QPA_PLATFORM=offscreen`；CI fresh runner 只
安装 JRE 和 `xvfb`，不会安装 Python 或项目依赖。

## Windows 提示缺少 DLL

请使用同一 ZIP 内完整 onedir 目录，不要只复制其中的 EXE。若 onefile 与
onedir 都失败，保留 `--self-check --json-report` 的 JSON 和控制台日志。不要从
其他 Python 或 MSYS2 目录手工混入 DLL，这会掩盖发布包缺库问题。

## 任务显示“结果已过期”

这是 revision/dependency 发布门禁生效：任务运行期间源码或 import 依赖发生了
变化。确认当前源码有效后重新执行。过期结果不会覆盖当前文档或发布截断产物。

## 动态用例不一致

报告中的 `expected`、`actual`、`diffs`、`scenario_sha256`、模型 fingerprint 和
source revision 用于定位差异。内置资源 SHA 不匹配时应恢复仓库固定字节，不能
通过忽略 CRLF/LF 差异来绕过 provenance 检查。

## 提交问题所需证据

至少附上系统架构、Java 版本、onedir/onefile 类型、self-check JSON、acceptance
JSON、失败项日志和对应截图。默认报告已脱敏；主动显示完整路径后应再次检查再
公开。
