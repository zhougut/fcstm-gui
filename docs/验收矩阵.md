# 发布验收矩阵

## 两阶段平台门禁

| 阶段 | Linux x86_64 | Windows x86_64 | macOS x86_64 |
|---|---:|---:|---:|
| 源码 pytest + self-check + GUI acceptance | 必须 | 必须 | 必须 |
| onedir 构建后完整检查 | 必须 | 必须 | 必须 |
| onefile 构建后完整检查 | 必须 | 必须 | 必须 |
| fresh runner onedir 黑盒检查 | 必须 | 必须 | 必须 |
| fresh runner onefile 黑盒检查 | 必须 | 必须 | 必须 |

fresh runner 不 checkout、不 setup Python、不执行 pip，只下载上一阶段产物并配置
Java；Linux 另配 `xvfb`。六个 matrix job 必须全部成功。

## self-check 核心项

- pyfcstm 全模块 closure，另有真实 loader、inspect、CLI、渲染和高亮行为。
- Z3 独立执行 integer SAT/model、UNSAT、exact real、bit-vector、Optimize 五项。
- 公式 logical/numeric/effect/lifecycle 各有合法与非法项。
- 普通仿真覆盖初始化、多 cycle、结束、异常 cause 与 rollback。
- 四个上游动态用例各自独立执行，另有 mutation mismatch、恢复重跑和 8 个资源 SHA。
- 五模板逐一 inventory/info/extract/render；build 阶段额外执行 Python 产物并编译运行 C/C poll/C++/C++ poll。
- Qt、QScintilla、qtawesome、qtmodern、XLSX、DOCX、PlantUML Java 子进程均执行真实逻辑。

## GUI acceptance 11 项

1. `workflow.document-open`
2. `workflow.model-edit-undo-redo`
3. `workflow.structured-diagnostics`
4. `workflow.graph-refresh`
5. `workflow.ordinary-simulation`
6. `workflow.dynamic-validation`
7. `workflow.code-generation`
8. `workflow.unified-export`
9. `workflow.task-results`
10. `workflow.failure-recovery`
11. `geometry.viewport-accessibility`

每个源码态、onedir、onefile 与 fresh-runner 产物均覆盖 1280×720、1920×1080
和缩放 1、1.5、2 的交叉组合，并上传 versioned JSON、日志、截图和导出产物。

## 声明边界

- 动态验证不是形式化验证。
- 当前矩阵不构成 Windows 7 实机兼容性声明。
- build job 成功不能替代 fresh runner；任一 JSON item、SHA provenance、产物上传
  或 fresh job 失败，整次发布验收即失败。
